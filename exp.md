# exp0-baseline

Референсний (baseline) експеримент. Нічого не унаслідовано — це перша гілка стеку.

## Що змінено в цьому експерименті

1. **Реверт мерджу PR RTDT-7618** (`9bb25de`) — повернення до старого формату датасету:
   - `img_folder: /dataset/train/` (train) та `/dataset/test/` (val)
   - `ann_file: /dataset/coco_annotations.json` (спільний для train/val)
   - відповідно ревертнуто `inference.py` та один рядок у `deim_criterion.py`.

2. **`profile_sync` — правдивий профайлінг** (`engine/solver/det_engine.py`,
   `engine/core/_config.py`, `engine/solver/det_solver.py`, `configs/runtime.yml`):
   CUDA-кернели асинхронні, тому старі таймери міряли постановку кернелів у чергу,
   а GPU-час «спливав» на прихованих точках синхронізації і приписувався чужим
   стадіям (саме тому «Criterion 29%» у старих логах не можна брати за чисту монету).
   З `-u profile_sync=True` перед кожним зчитуванням таймера викликається
   `torch.cuda.synchronize()` — атрибуція стадій стає правдивою. За замовчуванням
   вимкнено (нуль overhead у продакшн-тренуваннях). Оркестратор вмикає його завжди.

3. **`stg1_epochs_perc` override** (`tools/utils.py`): частку no-aug стадії можна
   задати через `-u stg1_epochs_perc=0.5` (дефолт 1/6 не змінився). Для 6 епох
   дефолт дає рівно 5+1 (межа = int(5/6·6) = 5).

4. **`run_experiments.sh`** — оркестратор: послідовно запускає гілки-експерименти
   в окремих контейнерах (детальніше в `speedup.md`, розділ «Крок 0»).

5. **`tools/benchmark_decode.py`** — офлайн-бенчмарк бекендів JPEG-декодування
   (PIL vs OpenCV vs torchvision.io vs PyTurboJPEG). Запустити один раз на сервері
   ДО тренувань: якщо переможець дає >1.5× на декоді — має сенс гілка
   `exp5-decode-backend` (див. `speedup.md`, «Крок 0.5»).

## Вплив на тренування

Нульовий: жодних змін швидкості чи семантики порівняно з main (окрім реверту
формату датасету). Цей ран задає базову правдиву таблицю Profiling/* у TensorBoard.

## Як запускати

```bash
./run_experiments.sh exp0-baseline
```

## На що дивитися в TensorBoard

- `Profiling/time_*` та `Profiling/pct_*` по епохах — нова правдива атрибуція стадій.
  Очікування: розподіл зміститься відносно старих логів (частина «Criterion» піде
  у «Forward», бо тепер forward чекає завершення своїх кернелів).
- Епоху 0 виключати з порівняння (прогрів: старт воркерів, кеші).
- `Test/*` — mAP по епохах (sanity, не метрика якості на 6 епохах).
